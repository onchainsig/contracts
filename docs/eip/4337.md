# Abstract Account

## 背景

钱包类型
- 私钥管理的钱包
- 合约钱包：zkSync, starkNet 等原生支持 AA
- MPC 钱包：SSS 和 TSS 两种方式，TSS 是趋势

EOA -> Contract Account -> Abstract Account

AA 抽象的到底是什么？

交易类型：Legacy, EIP2718, EIP1559
账号结构：

定义 from 方

资产所有权和签名权耦合

1. 私钥难保护
2. 签名算法少
3. 签名权限高 - 没有原生多签方案（多签需要用智能合约实现）
4. 单币种结算
5. 交易隐私低
